## What ARVI Is

ARVI (Agentic Regeneration Via Intelligence) is an autonomous environmental monitoring agent for urban forests in Mexico City.

Users deploy physical sensors in parks and forests. Each sensor streams hyperlocal data — CO₂, soil moisture, biodiversity signals, air quality — that satellites and city APIs cannot capture. Venice AI analyzes these streams continuously and acts on anomalies: alerting operators, logging events on Base, and posting open missions for other agents.

The core value: **passive income for citizens who deploy sensors + real-time environmental intelligence for cities and NGOs — fully autonomous, no human trigger required.**

## How It Works (The Real Loop)

```
Open-Meteo API (real weather data)
        ↓
Venice AI — llama-3.3-70b (private inference, zero retention)
        ↓
Anomaly detected? → Email alert via Resend
        ↓
Alert written to Cloudflare R2 (public, permanent)
        ↓
Event emitted on Base Mainnet via ARVIAgent contract
```

This loop runs automatically every hour via Vercel cron. Zero human trigger.

**Try it:**
```bash
# Full traceable chain — one command:
curl https://arvi-eight.vercel.app/api/demo | jq .

# Direct analysis:
curl -X POST https://arvi-eight.vercel.app/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"node_id": "node-01"}'
```

## Evolution — Rembu → Sen → Aona → ARVI

**Rembu (v1):** First attempt at a sensor marketplace. Focused on connecting IoT devices to a dashboard. No autonomous agent — just data visualization.

**Sen (v2):** Added AI analysis layer. Tried to connect multiple LLM providers (including Bankr, which turned out to be a token launchpad with no usable API, and OpenServ, which had no real integration path). Realized the stack was overbuilt and underconnected.

**Aona (v3):** Pivoted to agentic framing — the agent takes actions, not just shows data. Added Base contract, ERC-8004 identity, Resend email alerts. Tried Locus for payments (API not publicly accessible). Removed fake integrations.

**ARVI (v4 — current):** One clean loop, one honest stack. Venice AI is the only LLM. Payments via ARVIAgent on Base. Alert log on Cloudflare R2 (public, permanent). Every artifact is verifiable.

## What Was Evaluated and Removed

These were explored during development. They are not in the codebase, not in the submission narrative, and not in the sponsor tracks:

| Service | Why removed |
|---------|-------------|
| Bankr | Turned out to be a token launchpad. `api.bankr.ai/v1` returns nothing useful. |
| OpenServ | No functional API integration path within the hackathon scope. |
| Locus | Evaluated for USDC payments to operators. API not publicly accessible. Payments now handled by ARVIAgent on Base. |
| EVVM | Explored for gasless infra. Not integrated. |
| Status Network | Evaluated for gasless node operator onboarding. No integration completed. |
| Markee | Evaluated for data stream access control. Not integrated. |

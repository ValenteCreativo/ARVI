## What ARVI Is

ARVI (Agentic Regeneration Via Intelligence) is a decentralized network of autonomous environmental agents — deployable anywhere on the planet.

Citizens deploy physical sensors in forests, wetlands, rivers, coastlines, urban parks, or any ecosystem. Each sensor generates hyperlocal environmental intelligence — CO₂, soil moisture, biodiversity signals, air quality, pathogen risk — that satellites and government APIs cannot capture at this resolution or speed. Venice AI analyzes these streams continuously, detects threats, and acts: alerting operators, logging immutable events on Base, and posting open missions for other agents to coordinate.

The network is self-sustaining: sensor operators earn passive income for their contribution. The more sensors, the more intelligence. The more intelligence, the more value to cities, NGOs, impact projects, and other agents.

The current deployment is in Mexico City — Chapultepec, Xochimilco, Alameda Central — as proof of concept. The architecture is global by design.

The core thesis: **environmental data should be decentralized, incentivized, and acted upon autonomously — not locked in government databases waiting for a human to notice.**

## How It Works (The Real Loop)

```
Open-Meteo API (real weather data)
        ↓
Venice AI — llama-3.3-70b (private inference, zero retention)
        ↓
Anomaly detected? → Email alert via Resend
        ↓
Alert written to Cloudflare R2 (public, permanent)
        ↓
Event emitted on Base Mainnet via ARVIAgent contract
```

This loop runs automatically every hour via Vercel cron. Zero human trigger.

## Business Model

ARVI is self-sustaining because it has multiple revenue streams that fund operator rewards and network growth. This is not a grant-dependent project.

### Revenue Streams

**1. Data as a Service (DaaS)**
Cities, NGOs, researchers, and corporations pay for access to hyperlocal environmental data streams. This is the core revenue engine. A city paying $500/month for 10 nodes covering a park is already a viable unit — and ARVI's data is more granular and real-time than anything a government agency provides.

**2. Hardware — Sensor Sales**
Citizens and organizations buy sensors to deploy and earn passive income. The sensor is the entry point: buy once, earn continuously based on data quality and uptime. Margin on hardware funds initial network expansion.

**3. Enterprise ESG Compliance**
Corporations with environmental commitments (carbon neutrality, biodiversity targets, supply chain sustainability) need continuous, third-party-verifiable data. ARVI provides exactly that — sensor-backed, AI-analyzed, timestamped, and logged on Base. This is not self-reported sustainability. It's verifiable evidence for ESG auditors and regulators.

Target: $2,000–$10,000/month per enterprise client for continuous monitoring contracts.

**4. Impact Project Evaluation (Octant / Gitcoin / Carbon Credits)**
Impact projects need MRV (Monitoring, Reporting, Verification) to access carbon credits and impact funding. ARVI provides the sensor infrastructure and AI analysis layer. Projects pay per evaluation cycle.

This is the Octant integration: any project funded by Octant can deploy an ARVI node and get objective, sensor-backed impact scores instead of self-reported claims.

**5. Periodic Reports — NGOs and Municipalities**
Monthly or quarterly environmental intelligence reports for city governments and conservation NGOs. Formatted for non-technical audiences. Priced as a subscription.

Target: $300–$1,500/month per organization.

**6. Agent Inference as a Service (x402)**
Other AI agents pay ARVI per query for environmental intelligence. Priced in USDC via x402 micropayments on Base. Discoverable via `/arvi.skill.md`. As the agent economy grows, this becomes a passive revenue stream that scales without human sales effort.

### Unit Economics (Realistic Projection)

| Metric | Value |
|--------|-------|
| Sensor cost (hardware) | ~$80–$120 USD |
| Operator reward model | Variable — per data point confirmed + uptime bonus |
| Early network (< 1,000 nodes) | $2–$8/month per node (scarcity premium) |
| Mature network (10,000+ nodes) | $0.50–$2/month per node (volume drives total) |
| City data subscription (10 nodes) | $300–$500/month |
| Enterprise ESG contract | $2,000–$10,000/month |
| Break-even per node | 12–24 months (not a get-rich-fast scheme) |
| Margin per enterprise contract | 60–70% |

Operator rewards are variable, not fixed. They depend on:
- **Data quality score** — accuracy and calibration vs. reference stations
- **Uptime %** — continuous availability earns more than sporadic data
- **Location scarcity** — nodes in uncovered ecosystems command a premium
- **Query volume** — nodes in high-demand areas earn more per month

This is intentionally modest at scale — the value proposition for operators is long-term passive income from infrastructure they own, not a high-yield speculation. Think Helium for environmental data: individual rewards are small, but the network effect is the asset.

The flywheel: more sensors → more coverage → more enterprise contracts → more revenue → more rewards distributed → more operators deploy sensors.

### Why Operators Deploy Sensors

The passive income model solves the deployment problem that killed every previous environmental monitoring network: **who pays for the sensors and who maintains them?**

In ARVI, the answer is: citizens who want passive income. The network deploys itself because there's a direct economic incentive for each individual node operator — independent of grants, government budgets, or corporate sustainability programs.

## Research Context

ARVI is the fourth iteration of a Master's thesis in Strategic Business Administration (MBA) focused on technology for environmental impact. The research methodology is experiment-based: each version is deployed in a real hackathon, learnings are documented, and the next version integrates what was validated.

This is not a pivot story. It is a research arc.

---

## Evolution — Rembu (2024) → SEN (2025) → Aona (2025) → ARVI (2026)

**Rembu (2024) — DePIN Experiment**
Research question: *Can decentralized physical infrastructure (DePIN) be used to incentivize environmental sensor deployment?*
Result: Validated that economic incentives can drive citizen sensor adoption. Identified the gap: data was being collected but not analyzed or acted upon. Documented learnings fed directly into SEN.

**SEN (2025) — Data Commercialization Experiment**
Research question: *Can environmental sensor data be commercialized as a service — creating a market that sustains the network?*
Result: Validated demand for hyperlocal environmental data from cities, NGOs, and researchers. Identified the bottleneck: human analysts were the limiting factor in turning data into action. Documented learnings fed directly into Aona.

**Aona (2025) — AI Integration Experiment**
Research question: *Can AI analysis remove the human bottleneck — turning raw sensor data into structured, actionable intelligence automatically?*
Result: Validated AI-driven environmental analysis. Confirmed that pattern detection and anomaly identification can be automated. Identified the next frontier: the AI could *analyze* but could not yet *act*. Documented learnings fed directly into ARVI.

**ARVI (2026) — Agentic Action Experiment**
Research question: *Can autonomous AI agents close the full loop — from environmental signal to real-world action — with zero human trigger?*
Result (current): The loop is proven. Venice AI analyzes in real time. Alerts are sent. Events are logged on Base. The agent runs hourly without human intervention. The next frontier: expanding the action set (field APIs, operator payments, government notifications) using the same architecture already validated.

## Why This Matters as a Research Contribution

Each experiment added one layer to the same system:
1. **Rembu** — the incentive layer (why people deploy sensors)
2. **SEN** — the value layer (what the data is worth)

ARVI does not replace its predecessors. It integrates their validated learnings into a working system. The codebase is the thesis output — not a prototype, not a demo, but a tested, deployable tool for environmental resilience.

### ENS — Agent Identity Infrastructure

ENS is the identity layer for node operators and agent participants in ARVI's network. The integration is technical, not cosmetic.

**Live ENS resolution endpoint:**
```bash
curl https://arvi-eight.vercel.app/api/ens?name=vitalik.eth
```

Returns: `{ resolved: true, address: "0xd8dA...", source: "ensdata.net" }`

**How ARVI uses ENS:**
1. Every sensor node has an ENS identity in the node registry (e.g., `n1.arvi.eth`, `n2.arvi.eth`)
2. When `/api/analyze` runs, it resolves the operator's .eth name before routing the alert
3. The `ens_resolved` field appears in every analysis response — name + resolved address
4. Operator payments on Base route to the resolved address, not to a hardcoded wallet

**Why this matters for the track:**
Node operators are not crypto-native. Asking them to manage raw wallet addresses creates friction. ENS lets them register a human-readable identity once and receive sensor rewards, alerts, and mission assignments by .eth name — across any chain, any interface, any agent.

This is ENS as payment infrastructure, not as a naming novelty.

## Technical Roadmap — Next Infrastructure Layer

### EVVM — Virtual Blockchain over Base

EVVM (evvm.org) allows deploying a virtual blockchain via smart contracts on top of any existing network. For ARVI, the target deployment is Base Mainnet. This gives the network:

- **Gasless transactions** — node operators participate without needing ETH for gas. Critical for onboarding non-crypto-native environmental stewards.
- **Full operational sovereignty** — ARVI's economic layer runs on its own virtual chain, isolated from gas market volatility
- **ENS-compatible NameService** — native naming for operators and agents, without ENS registration fees
- **Built-in staking, treasury, and P2P swap** — the 6 EVVM contracts cover the full economic infrastructure ARVI needs

**What was done:**
During development, all 6 EVVM contracts were deployed and validated in a local Foundry/Anvil environment:

This is not a pivot. It is the next layer of the same architecture.

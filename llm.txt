## What ARVI Is

ARVI (Agentic Regeneration Via Intelligence) is an autonomous environmental monitoring agent for urban forests in Mexico City.

Users deploy physical sensors in parks and forests. Each sensor streams hyperlocal data — CO₂, soil moisture, biodiversity signals, air quality — that satellites and city APIs cannot capture. Venice AI analyzes these streams continuously and acts on anomalies: alerting operators, logging events on Base, and posting open missions for other agents.

The core value: **passive income for citizens who deploy sensors + real-time environmental intelligence for cities and NGOs — fully autonomous, no human trigger required.**

---

## How It Works (The Real Loop)

```
Open-Meteo API (real weather data)
        ↓
Venice AI — llama-3.3-70b (private inference, zero retention)
        ↓
Anomaly detected? → Email alert via Resend
        ↓
Alert written to Cloudflare R2 (public, permanent)
        ↓
Event emitted on Base Mainnet via ARVIAgent contract
```

This loop runs automatically every hour via Vercel cron. Zero human trigger.

**Try it:**

## Research Context

ARVI is the fourth iteration of a Master's thesis in Strategic Business Administration (MBA) focused on technology for environmental impact. The research methodology is experiment-based: each version is deployed in a real hackathon, learnings are documented, and the next version integrates what was validated.

This is not a pivot story. It is a research arc.

---

## Evolution — Rembu (2024) → SEN (2025) → Aona (2025) → ARVI (2026)

**Rembu (2024) — DePIN Experiment**
Research question: *Can decentralized physical infrastructure (DePIN) be used to incentivize environmental sensor deployment?*
Result: Validated that economic incentives can drive citizen sensor adoption. Identified the gap: data was being collected but not analyzed or acted upon. Documented learnings fed directly into SEN.

**SEN (2025) — Data Commercialization Experiment**
Research question: *Can environmental sensor data be commercialized as a service — creating a market that sustains the network?*
Result: Validated demand for hyperlocal environmental data from cities, NGOs, and researchers. Identified the bottleneck: human analysts were the limiting factor in turning data into action. Documented learnings fed directly into Aona.

**Aona (2025) — AI Integration Experiment**
Research question: *Can AI analysis remove the human bottleneck — turning raw sensor data into structured, actionable intelligence automatically?*
Result: Validated AI-driven environmental analysis. Confirmed that pattern detection and anomaly identification can be automated. Identified the next frontier: the AI could *analyze* but could not yet *act*. Documented learnings fed directly into ARVI.

**ARVI (2026) — Agentic Action Experiment**
Research question: *Can autonomous AI agents close the full loop — from environmental signal to real-world action — with zero human trigger?*
Result (current): The loop is proven. Venice AI analyzes in real time. Alerts are sent. Events are logged on Base. The agent runs hourly without human intervention. The next frontier: expanding the action set (field APIs, operator payments, government notifications) using the same architecture already validated.

## Why This Matters as a Research Contribution

Each experiment added one layer to the same system:
1. **Rembu** — the incentive layer (why people deploy sensors)
2. **SEN** — the value layer (what the data is worth)
3. **Aona** — the intelligence layer (what AI can extract from data)
4. **ARVI** — the action layer (what agents can do with intelligence)

ARVI does not replace its predecessors. It integrates their validated learnings into a working system. The codebase is the thesis output — not a prototype, not a demo, but a tested, deployable tool for environmental resilience.

### Octant — Data Collection & Analysis for Project Evaluation
Impact projects often claim environmental outcomes that are impossible to verify.
ARVI provides the infrastructure to change that: continuous, sensor-backed, AI-analyzed environmental data. Any Octant-funded reforestation or wetland project can deploy an ARVI sensor and get objective, verifiable impact scores — not self-reported claims.

### ENS — Agent Identity Infrastructure
ENS is not used so ARVI can have a pretty name. It's used as identity infrastructure for the agents and node operators that interact with the network.

Node operators register .eth names. Agents query environmental intelligence using .eth identifiers instead of raw wallet addresses. This enables:
- Operator identity: `chapultepec-a.arvi.eth` → consistent across deployments, human-verifiable
- Agent-to-agent lookup: any agent that wants to pay or coordinate with an ARVI node uses ENS resolution, not raw addresses
- Payment routing: operator rewards on Base resolve to .eth names first, then to wallet

This is infrastructure for the multi-agent economy ARVI is building — not cosmetic branding.

### Synthesis Open Track
ARVI addresses the gap between environmental urgency and agentic infrastructure. Submitted to the Synthesis Open Track as a demonstration of what autonomous agents can do for planetary resilience.

## What Was Evaluated and Removed

These were explored during development. They are not in the codebase, not in the submission narrative, and not in the sponsor tracks:

| Service | Why removed |
|---------|-------------|
| Bankr | Turned out to be a token launchpad. `api.bankr.ai/v1` returns nothing useful. |
| OpenServ | No functional API integration path within the hackathon scope. |
| Locus | Evaluated for USDC payments to operators. API not publicly accessible. Payments now handled by ARVIAgent on Base. |
| EVVM | Explored for gasless infra. Not integrated. |
| Status Network | Evaluated for gasless node operator onboarding. No integration completed. |
| Markee | Evaluated for data stream access control. Not integrated. |
